# 간이 주문·재고 관리 시스템 (Order & Inventory Management)

## 1. 프로젝트 목적

이 프로젝트는 프론트엔드 개발자로서 React/Next.js 기반 개발 경험을 바탕으로,  
**기업용 시스템(ERP/WMS)에서 요구되는 백엔드 아키텍처와 비즈니스 로직을 Java(Spring Boot)로 구현해보는 것**을 목표로 한다.

특히 단순 CRUD를 넘어,

- 명확한 도메인 규칙
- 상태 전이(State Transition)
- 트랜잭션 처리
- Service 레이어 중심의 비즈니스 로직 설계

를 직접 설계하고 구현함으로써,  
**엔터프라이즈 백엔드 개발에 필요한 사고방식과 구조**를 학습하는 것을 목적으로 한다.

---

## 2. 이 프로젝트를 통해 반드시 학습해야 할 핵심 개념

### ✔ 엔티티 간 관계 설계
- 주문(Order), 상품(Product), 재고(Inventory) 간의 관계를 명확히 정의
- 단순 테이블 연결이 아닌 **도메인 관점에서의 연관 관계 이해**

### ✔ 상태 전이(State Transition)
- 주문은 생성 → 확정 → 출고 완료 / 취소 등의 상태를 가짐
- 특정 상태에서는 허용되지 않는 행위가 존재함

### ✔ 트랜잭션(Transaction)
- 주문 생성 시 재고 차감
- 주문 취소 시 재고 복구
- 여러 엔티티 변경이 **하나의 논리적 작업 단위로 처리**되어야 함

### ✔ Service 레이어 중심의 비즈니스 로직
- Controller는 요청/응답만 담당
- 비즈니스 규칙은 Service 계층에 집중
- Repository는 데이터 접근 책임만 가짐

### ✔ ERP / WMS 스타일 아키텍처
- 사용자 편의보다 **데이터 정합성과 규칙의 명확성**을 우선
- 실무 ERP/WMS 시스템과 유사한 구조와 책임 분리

---

## 3. 서비스 개요

본 서비스는 **간이 주문·재고 관리 시스템**으로,  
상품의 재고를 관리하고 주문 생성, 출고, 취소 등의 흐름을 관리한다.

### 주요 기능
- 상품 등록 및 재고 관리
- 주문 생성 및 주문 상태 관리
- 출고 처리
- 주문 취소 처리

이 서비스는 B2C 서비스가 아닌,  
**기업 내부에서 사용하는 관리 시스템(ERP/WMS)** 을 가정하여 설계된다.

---

## 4. 핵심 도메인 규칙

### ✔ 재고 부족 시 주문 불가
- 주문 생성 시 상품 재고 수량이 부족하면 주문을 생성할 수 없다.
- 단순 UI 제어가 아닌 **백엔드 도메인 규칙으로 강제**한다.

### ✔ 출고 완료 후 주문 수정 불가
- 출고가 완료된 주문은 수정하거나 취소할 수 없다.
- 주문 상태에 따른 행위 제한을 명확히 한다.

### ✔ 주문 취소 시 재고 복구
- 주문이 취소되면 차감되었던 재고는 원래 수량으로 복구된다.
- 이 과정은 트랜잭션으로 처리되어야 한다.
